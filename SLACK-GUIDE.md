# Rehearsal Social — Slack Operator Guide

A plain-English guide to running our social posting from Slack. No technical setup, no
code. If you can send a Slack message and click a button, you can run this.

Everything happens in the **#rehearsal-social** channel. The assistant that helps you
there is called **Hermes**.

---

## The one thing to remember

> **You are always in control. Nothing posts to LinkedIn, X, or Instagram until you click a Publish button (or schedule it). Hermes never posts on its own.**

The job is simple: **pick a ready-made post → look it over → publish it (or schedule it).**

---

## The everyday flow (start here)

### 1. See what's ready to post

Type this in the channel and press enter:

```
/posts
```

You get a list of every finished post in our library. Each one shows:

- **The title**
- **The platform** — 🔗 LinkedIn, ✖️ X, or 📸 Instagram
- **The theme** (pillar) and a **suggested time slot**, if set
- Little flags like `🧵 8-tweet thread` or `⚠️ needs image to publish`

Each post has a **Select** button.

### 2. Open a post for review

Click **Select** on the post you want. Hermes posts a **review card** with:

- The **hook** (the opening line)
- The **full caption** exactly as it will publish
- An info line: character count, how many images, whether it's a thread, and whether
  it came **📚 from content library** (ready-made)

### 3. Choose what to do

The review card has buttons. The common ones:

| Button | What it does |
|---|---|
| ✅ **Approve** | Marks the post as reviewed and good to go. |
| ✏️ **Edit** | Opens a box to tweak the caption text before publishing. |
| 🔎 **Re-evaluate** | Asks Hermes to check the post against our brand voice and flag anything off. |
| 🗓️ **Schedule** | Pick a date and time; it publishes automatically then (see below). |
| 🟦 **Publish → LinkedIn / X / Instagram** | Publishes **now**, to the right platform. |
| 🔁 **Adapt for …** | Rewrites the post to fit the *other* platform (e.g. turn a LinkedIn post into an X version) — never a copy-paste. |

> **You only ever see the Publish button for the post's own platform.** An X post can't
> be accidentally published to LinkedIn, and vice-versa. That's on purpose.

### 4. Publish or schedule

- **Publish now:** click the blue **Publish → …** button. Done — it's live.
- **Schedule:** click **🗓️ Schedule**, pick a date and time, confirm. The system
  publishes it for you at that moment, even if you're offline. You don't need to keep
  Slack open.

That's the whole loop.

---

## Talking to Hermes (just type, no tagging)

You can chat with Hermes in the channel like a teammate. **You don't need to tag it** —
just write your message, or reply in a thread to keep a conversation going. Ask things like:

- "Which platform fits this better — X or LinkedIn?"
- "What's scheduled this week?"
- "Is this caption on-brand?" (paste the caption)
- "Remind me what our voice rules are."

Hermes answers in plain language. It will **help you think and check things** — but it
won't write or publish posts for you. That stays human.

---

## The safety nets (so you can relax)

These run automatically. You don't have to do anything — they just stop mistakes:

- **No wrong-platform posts.** You only get the Publish button for the post's own platform.
- **No empty Instagram posts.** Instagram (and any image-essential post) **can't be
  published without an image** — the card will say *"🖼️ Needs an image to publish"* and
  hide the Publish button until one is added. You can still Schedule it for later.
- **No double-posting.** If a post was already published, the system won't publish it
  again — even if a button is clicked twice or the system restarts.
- **No em dashes.** Our brand rule is no "—" in published copy; this is enforced for you.
- **Threads stay whole.** An X thread (e.g. 8 tweets) publishes in full, in order — never
  chopped down to the first line.

## A couple of things to know right now

- **Four X posts are waiting on designed images.** Some X posts in the library are
  card- or thread-style and need a proper designed visual (the social/design team will
  create these). Until an image is added, the system **won't let them publish text-only** —
  that's the safety net doing its job, not a bug. Posts that are fine as text (like the
  Karnataka X post) publish normally.
- **`/draft` is off in this version.** You may see a `/draft` command — it's for AI-writing
  new posts and is **not enabled**. Our version is about distributing the finished posts we
  feed into the library, not generating new ones. Stick with `/posts`.

## Quick reference

| I want to… | Do this |
|---|---|
| See what's ready to post | Type `/posts` |
| Review a specific post | Click **Select** next to it |
| Tweak the wording first | **✏️ Edit** on the card |
| Double-check it's on-brand | **🔎 Re-evaluate**, or ask Hermes in chat |
| Post it right now | **🟦 Publish → [platform]** |
| Post it later | **🗓️ Schedule** → pick date/time |
| Make an X version of a LinkedIn post (or vice-versa) | **🔁 Adapt for …** |
| Ask a question | Just type it in the channel |

---

## When something looks wrong

- If a post won't show a Publish button → check the card for a **"needs an image"** note.
  That post needs a visual before it can go out.
- If a command doesn't respond, give it a few seconds and try once more.
- If you're unsure whether something published, type `/posts` or ask Hermes
  "what went out today?" before re-clicking — the double-post guard protects you, but
  it's good to confirm.
- For anything stuck, ping the person who set this up (the system owner). You won't break
  anything by asking.

---

*That's it. The rhythm is: `/posts` → **Select** → look it over → **Publish** or
**Schedule**. Everything else is there to help you, not to get in your way.*
